Before provisioning volumes, every storage node needs a MiroirNode custom
resource, applied separately from this chart (kubectl, GitOps, ...):
  apiVersion: miroir.home-operations.com/v1alpha1
  kind: MiroirNode
  metadata:
    name: <node-name>
  spec:
    pools:
      - name: default
        lvmthin:                        # or zfs / loopfile — the block IS the backend
          device: /dev/disk/by-partlabel/r-miroir
        # zfs: { dataset: pool/miroir }
        # loopfile: { baseDir: /var/lib/miroir }  # reflink-capable FS
Agents on nodes without a MiroirNode serve client-only (RWX mounts) and
switch to storage mode by themselves when their MiroirNode appears.
StorageClasses select a pool with their `pool` knob (absent = "default").
Inspect the topology with `kubectl get miroirnodes`.

Verify the rollout:
  kubectl -n {{ .Release.Namespace }} get pods -l app.kubernetes.io/name={{ include "miroir.name" . }}
  kubectl -n {{ .Release.Namespace }} get daemonset {{ include "miroir.agentName" . }}
  kubectl get csidriver {{ include "miroir.csiDriverName" . }}
